176 /*
177  AnimationController 設計方針 Note:
178  - Animator は、Component の派生。これはボーンアニメーションに限らず、アタッチ先の WorldObject 及び子 Components のプロパティにバインドする。
179  - AnimationController は、Object の派生。通常 SkinnedMeshModel と 1:1 で作ることになる (Three.js の AnimationMixier のイメージ)
180  これらの違いはプロパティのバインド先がプロパティかボーンorモーフかだけとしたい。
181 
182  ゲーム作るときほとんどの場合は Animator を使う。SkinnedMeshComponent がボーンをプロパティとして公開し、それにバインドする形になる。
183 
184  AnimationController は SkinnedMeshModel を直接コントロールするときに使う。SkeltalAnimationController とか名前変えてもいいかな。
185  Animator と比べて余計なプロパティ検索しないので軽量。ただし Editor からいじることはできない。
186  Component は使わず SkinnedMeshModel だけを使って描画したいときや、Component の中だけで完結させたい (パーティクルに SkinnedMesh を使うとか？) 時に使う。
187 
188 
189  使い方Note:
190  SkinnedMesh の場合
191  - アタッチしたとき、Bone の分だけ Link を作って addElementBlendLink() しておく。dataId には BoneIndex を入れておく。
192  - 後は addClip() されたときに内部で、この登録された Link を名前検索して Trask と紐づけられ、advanceTime() で値が更新される。
193  - 更新後、Link を for して dataId に対応する Bone 、Link.value をセットする。
194  プロパティアニメの場合
195  - Bone と違ってどのようなプロパティが WorldObject に現れるかわからないし、Dynamic に変わったりする。
196  実際のところ Transform のアニメがほとんどで、他のプロパティはアニメの必要が無いものも多い。全部 Link 作るとメモリ効率悪い。
197  - addClip() されたときに、Clipが必要としているプロパティを検索して、ヒットすれば Link を作る方がいいかもしれない。
198  SkinnedMesh 考え直し
199  - プロパティアニメと同じ方法にしておこうか。やり方増やすとメンテ大変だし、実行速度としてもそれほど変わるものではない。
200 */
