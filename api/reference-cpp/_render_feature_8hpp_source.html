24 // Rendering モジュールの後段の出力結果。
25 // RenderingContext(DrawElementListBuilder) によって作られた DrawElementList は SceneRenderer に入力されると、
26 // Zソートやカリング・ステートのグループ化を考慮して RenderFeatureBatch のリストに変換される。
27 // 最終的には各 RenderFeature が RenderFeatureBatch を実行することで、GraphicsContext へ描画命令が渡る。
28 //
29 // RenderFeatureBatch が出来上がっている時点で、SpriteRenderer など Dynamic な VertexBuffer 等はすべて出来上がっている状態にしておく。
30 // RenderFeatureBatch 実行中は map や setData で、VertexBuffer や Texture 等への書きこみはできない。
31 // なぜこんな仕組みにしているかというと、特に Vulkan の RenderPass 実行中はデータ転送系のコマンドを実行できないため。
32 //
33 // RenderFeatureBatch は new してはならない。SceneRenderer から渡される LinearAllocator を使うこと。
34 //
